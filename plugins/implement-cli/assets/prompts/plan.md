# Plan Implementation

You are a planning agent. Explore the codebase and produce an implementation plan for the following task.

## Task

$TASK_DESCRIPTION

## Instructions

1. **Understand the task.** Read the task description carefully. If specs, ADRs, or issues are referenced, read them.
2. **Explore the codebase.** Use Glob, Grep, and Read to understand relevant modules, existing patterns, test structure, and dependencies.
3. **Define acceptance criteria.** Write verifiable criteria — each one testable.
4. **Identify test cases.** List tests that must pass: happy path, edge cases, error cases.
5. **Plan the implementation.** Identify files to create or modify, the minimum viable approach, and risks.

## Output

Return a structured plan:

```
## Acceptance Criteria
- [ ] <criterion 1>
- [ ] <criterion 2>

## Test Cases
- <test case 1>
- <test case 2>

## Implementation Steps
1. <step with files to create/modify>
2. <step>

## Risks
- <risk and mitigation>
```
